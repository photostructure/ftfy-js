## Lore

- `BADNESS_RE` is assembled in TS from the generated category class strings (mirror Python's
  verbose-regex `.format()`), verbose whitespace/comments stripped at build time by
  `stripVerbose()`, `u` flag set for correct Unicode codepoint ranges.
- `badness` uses `String.match(re).length` equivalent; `is_bad` uses `RegExp.test()`.
- **Regex `lastIndex` isolation**: A new `RegExp` instance is created on every call to
  `badness()` and `is_bad()` — since the compiled pattern source is a module-level constant,
  this is cheap (no string concatenation overhead in the hot path). This completely avoids the
  `lastIndex` contamination pitfall.
- **`u` flag**: Required for correct interpretation of multi-byte character class ranges
  (e.g. `═-╬` U+2550–U+256C, `Ё-Я` U+0401–U+042F, `А-Я`, box chars). Without `u`,
  these ranges would be interpreted as UTF-16 code units and could match incorrectly.
- **No `m` flag**: `^[ÃÂ][ ]` anchors to start of entire string, mirroring Python's
  `re.findall` without `re.MULTILINE`.
- **`stripVerbose()`**: A character-class-aware whitespace/comment stripper. Handles
  `#` comments only outside `[...]`, whitespace only outside `[...]`. Spaces inside
  character classes (e.g. `[ {start_punctuation}{end_punctuation}]`) are preserved.
- **Surrogate note**: No category ranges span the surrogate range (U+D800–U+DFFF), so
  the `u` flag is safe. The replacement character `�` is in the `bad` category but
  is well below surrogate territory.
- **`badness()` alone returns 0 for `�` (replacement char)**: This is correct Python
  behavior — `�` is in the `bad` category but triggers only in context (requires
  adjacent characters from other categories to form a match).

## Deliberate divergences from Python

1. **`RegExp` created per call, not cached as module-level singleton.** Python's
   `BADNESS_RE` is a compiled module-level object with no mutable state. JS `RegExp` with
   `g` flag carries `lastIndex`; rather than reset it on every call, we build from the
   constant source string. The cost is negligible; the safety guarantee is total.

2. **`sequence_weirdness()` is not ported.** The Python function is deprecated (emits a
   warning) and redirects to `badness()`. It has no upstream tests. The CLAUDE.md rule
   "faithful ≠ slavish" applies — no obligation to mirror deprecated internals.

3. **Python `\s`/`\w` are spelled as property classes (post-review fix).** JS `\w` is
   ASCII-only even under `u`, so the `[accented][punctuation]\w` alternative
   under-matched (Python `badness("é«é") == 1`, the original port gave 0 — found in
   review by codex and orchestrator independently). JS `\s` also differs (lacks
   `\x1c-\x1f`/`\x85`, adds `﻿`). The template now substitutes
   `{python_w}` → `[\p{L}\p{N}_]` and `{python_s}` → `[\t-\r\x1c-\x1f\x85\p{Z}]`,
   both verified exhaustively equal to CPython's `\w`/`\s` over all codepoints to
   U+10FFFF (unidata 15.0.0). Pinned by Python-derived test vectors (é«é, À»中, é«β,
   NEL/FS before upper-accented+currency, and a BOM negative case).
