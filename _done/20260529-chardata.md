## Key decisions / divergences

- **`Map` vs object for the data tables.** `CONTROL_CHARS`/`LIGATURES`/`WIDTH_MAP`/
  `HTML_ENTITIES`/`ENCODING_REGEXES` are `Map`s (codepoint-int or string keys),
  mirroring Python `dict`s; the translate-style consumers in Wave-3 `fixes.ts` iterate
  by codepoint and look up `Map.get`.
- **Global-flag split (the lastIndex pitfall).** Pure replace-all regexes
  (`SINGLE_QUOTE_RE`, `DOUBLE_QUOTE_RE`, `HTML_ENTITY_RE`, `LOSSY_UTF8_RE`) keep the
  `g` flag. The three dual-use regexes (`C1_CONTROL_RE`, `UTF8_DETECTOR_RE`, and ‚Äî
  per post-review fix ‚Äî `ALTERED_UTF8_RE`, which Python uses as `.search` in
  `__init__.py:510` AND `.sub` in `fixes.py:439`) are exported **without** `g` so
  `.test()` is a stateless predicate (matching the `utf8.ts` house convention);
  Wave-3 replace-all consumers must clone with `g` (e.g. `new RegExp(RE.source,
"gu")` ‚Äî but NO `u` for the byte-pattern `ALTERED_UTF8_RE`). Documented inline.
- **`possible_encoding` anchoring.** Compiled `^[...]*$` sources are tested with plain
  `RegExp.test` (Python `re.match`). Python's "`$` also matches before a final `\n`"
  rule is a no-op here because `\n` (0x0A) is a member of every encoding class
  (`\x00-\x19`); verified exhaustively against CPython over a 3-symbol alphabet (0
  diffs) and pinned by a `"x\n"`/`"\x80\n"` test.
- **Counts cross-checked against CPython 3.12 / unidata 15.0.0:** `HTML_ENTITIES`
  3501, `WIDTH_MAP` 226, `CONTROL_CHARS` 40, `LIGATURES` 22 ‚Äî all exact.

## Reviewer notes

- The byte regexes' literal `\xNN` escapes are code units 0x00‚Äì0xFF (binary-string
  bytes), compiled **without** `u`; a test confirms `ALTERED_UTF8_RE` does not match a
  U+01C3 char in real text (i.e. `\xc3` is the single unit 0x00C3, not an astral range).
- Whether the dual-use global-flag split (vs. shipping `g` everywhere) is the right
  contract for Wave-3 `fixes.ts` ‚Äî flagged so the fixers author knows to add `g` for
  the `C1_CONTROL_RE`/`UTF8_DETECTOR_RE`/`ALTERED_UTF8_RE` `.sub` sites.
- Post-review fix (codex + orchestrator vetting): `ALTERED_UTF8_RE` originally
  shipped with `g`, which made repeated `.test()` predicate calls skip matches
  (stateful `lastIndex`) ‚Äî a parity divergence from Python's stateless `.search` at
  `__init__.py:510`. Now non-global, pinned by a repeated-`.test()` test.
